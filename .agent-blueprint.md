# Digital Diva Academy — Technical Blueprint & Rules

This document serves as the "Source of Truth" for the Digital Diva Academy codebase. It is used by AI agents to maintain architectural consistency, visual integrity, and legal safety.

## 🏗 Core Architecture
- **Framework**: Next.js 16 (App Router)
- **Styling**: Tailwind CSS 4 (using CSS variables for design tokens)
- **Design Enforcement**: [DESIGN_AGENT.md](DESIGN_AGENT.md) (Creative Director & Accessibility rules)
- **Animations**: GSAP (Reveal.tsx) + React Lenis (Smooth Scrolling)
- **Rendering**: Server Components by default; Client Components for GSAP/Interactivity.
- **Global Layout**: `src/app/layout.tsx` handles:
    - Navbar (Automatic on all pages)
    - Footer (Automatic on all pages)
    - Sticky Disclaimer Banner (Top of page)
    - Smooth Scrolling/GSAP Registration

## 📂 Project Structure
- `src/app/`: Routes and Metadata.
- `src/components/common/`: Reusable primitive components (Button, Reveal, StatRow).
- `src/components/layout/`: Persistent frame (Navbar, Footer).
- `src/components/[category]/`: Domain-specific sections (legal/, credit/, offers/).
- `src/content/`: Structured data files (offers.ts, disclaimers.ts).
- `src/config/`: Centralized site settings and links.

## 🎨 Visual Identity
- **Palette**: Burgundy (#1c0408), Gold (#b68a48), Ivory (#faf3e8).
- **Typography**: 
    - Heading & Expressive: `Playfair Display` (Serif, weights 700/900)
    - Body: `DM Sans` (Sans-Serif, weight 400)
    - **Forbidden**: `Cormorant Garamond` (removed per brand update).
- **Design Language**: Boutique, cinematic, feminine but professional. High-contrast overlays and "artisan lighting" gradients.
- **Accessibility**: Strict WCAG 2.2 AA compliance on all layouts.

## 📜 Legal & Content Boundaries
- **Scope**: Legal education, credit empowerment, self-represented litigant coaching.
- **Hard Rules**:
    1. NEVER state that Vee is an attorney.
    2. NEVER promise specific court outcomes or credit score increases.
    3. Mandatory disclaimer must be visible on every page (handled by layout).
    4. Terminology: Use "Empowerment," "Education," and "Clarity." Avoid "Repair," "Fix," or "Representation."
